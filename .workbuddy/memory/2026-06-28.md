## 移动端选品指南 Bug 修复 (index.html)

### Bug 1: 确认加入弹窗飘走
- **问题**：qty-picker（确认加入数量选择框）在手机端滚动时会飘走
- **根因**：`position: fixed` 在移动端 WebView 中，滚动时定位不稳定
- **修复**：
  - CSS: 添加 `-webkit-transform: translateZ(0)` + `will-change: transform` 强制 GPU 加速
  - JS: 在 `showQtyPicker` 中添加 scroll/touchmove 监听（passive + once），一旦滚动即自动关闭选择器

### Bug 2: 产品卡片标签行高度不一致
- **问题**：金风习习 (4个标签) vs 绿扇盈盈 (1个标签) 导致同一行卡片高度不一致
- **根因**：`.p-tags-row` 的 `min-height: 20px` 不足以容纳 2 行标签
- **修复**：`min-height: 20px` → `min-height: 30px`（参考第四行 .p-desc 的 min-height 模式）

## 智能加载屏实现 (index.html)

### 方案
- 打开页面立即显示品牌加载屏（店名 + 悦刻品牌 + 金色进度条）
- 后台并行预加载所有产品图片 + 设备图片 + 二维码
- 图片全部就绪 或 最多 8 秒后 → 淡出加载屏，进入内容
- 最少展示 0.8s 避免闪烁（网速快时不白等）

### 改动
- CSS: 新增 `#loadingScreen` 完整样式（Teal 渐变背景 + 金色进度条 + 脉冲文字动画）
- HTML: 在 `<body>` 顶部插入加载屏 `<div id="loadingScreen">`
- JS: `renderAll()` 调用前插入 `initLoader()` IIFE，自动收集图片 / 预加载 / 控制过渡
- 加 12s 终极兜底 setTimeout，防止移动端 WebView 节流跳过 8s 定时器

## 导航高亮二次修复（三重保障）

问题仍存：
1. 网页+APP：点击后高亮的是上一区段（滚动略过露出底行导致）
2. APP（微信等）：点击后完全不亮

修复：
- **乐观高亮**：点击瞬间立即 `highlight(this)`，不等滚动结束（根治APP问题）
- **最近区段算法**：`offsetTop` 比较从"最后一个满足的"改为"距离最近的"（根治漂移问题）
- **双超时兜底**：`setTimeout(update, 600)` + `setTimeout(update, 1000)` 双重修正

## 导航高亮四次修复（scrollIntoView + scroll-padding-top）

- 原生锚点在逆序滚动时会轻微 overshoot（露出上板块底部）

- 新增 `html { scroll-padding-top: 120px }` 作为兜底
- 目标元素 `.section-header { scroll-margin-top: 120px }` 保留
- 点击乐观高亮 + scroll 事件修正手动滚动
