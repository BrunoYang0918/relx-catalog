## 选品清单图片生成 · 企微兼容修复（index.html）

### 问题根因
`exportCartPNG()` 用 `canvas.toDataURL()` + `link.click()` 下载图片。
企业微信内置浏览器**拦截 `a[download].click()`**，导致图片无法生成/下载。
个人微信浏览器对此限制较宽松，所以正常。

### 修复方案
放弃浏览器下载机制，改为**全屏预览弹窗**：
1. Canvas 绘制清单 → `toDataURL('image/png')`
2. 弹出 `.list-export-overlay` 全屏遮罩，显示生成的图片
3. 提示"长按图片即可保存到相册"（微信/企微均支持）
4. 底部备选按钮：`a[download].click()` 兜底（部分 WebView 可用）
5. 点背景/× 关闭
6. 关闭时恢复 body 滚动

### CSS 新增
- `.list-export-overlay` 全屏遮罩弹窗
- `.leo-img-wrap` 图片容器（白底圆角阴影）
- `.leo-hint` 保存提示文字
- `.leo-close` 关闭按钮
- `.leo-dl-fallback` 兜底下载按钮

## index.html · iOS 12 固定定位按钮无效 — 第二轮修复（ontouchstart 属性无效 → JS 监听器）

`ontouchstart=""` 属性方案无效（只能消除 300ms 延迟，无法解决 iOS 12 固定定位底部事件丢失）。

**新方案**：JavaScript `addEventListener('touchstart', ..., { passive: false })` 绕过
- 监听 `#floatingBar` 容器的 touchstart
- 用 `e.target.closest('.fb-btn')` 委托判断点击了哪个按钮
- `e.preventDefault()` 阻止系统吞事件
- 手动调用 `toggleGlossary()` / `openQuiz()` / `openCart()`
- 保留 inline onclick 作为现代浏览器正常路径
