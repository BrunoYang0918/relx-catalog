## i18n 第三轮修复（截图遗漏）

### 用户反馈（4张截图）
1. **口味词典(glossary)弹窗** — 全部中文，`<b>` 标签不渲染
2. **快速导航** — "烟弹选味"/"一次性款"/"烟具选色" 未翻译
3. **场景导航** — 按钮中文
4. **产品卡片desc** — 描述文字仍中文
5. **烟具卡片** — 颜色名(侠行山林等)/兼容性文字/规格特性 中文

### 根因分析
1. **Glossary**: i18n.js 对有子元素的 data-i18n 节点用 textNode.nodeValue 设值，HTML标签被当纯文本 → 修复为检测 dict 值含 `<` 时用 innerHTML
2. **快速导航 "一次性款"/"烟具选色"**: D 字典缺少 `qn_disposable_label` / `qn_devices_label` 键 → 已补充
3. **动态生成内容**: 烟具颜色名(c.label)、设备名(d.name)、设备特性(features)、产品描述(showDesc) 在 renderAll() 之后才写入 DOM，applyLang 的 section 5 回扫已执行完 → 用 __t() 包裹

### 修改文件
**i18n.js:**
- applyLang() 中 data-i18n 替换逻辑增加 HTML 检测：`dict[k].indexOf('<')!==-1` 时使用 `el.innerHTML`
- 补充 `qn_disposable_label:'Disposables'`, `qn_devices_label:'Devices'`

**index.html:**
- `renderDeviceSection()`:
  - `c.label` → `__t('dev_color_' + c.label, c.label)` (16个颜色名)
  - `d.name` → `__t(d.i18nKey, d.name)` (6个设备系列名)
  - compat/battery 前缀 → 英文 Compatible/Battery
  - `d.compat` → `__t('compat_' + d.compat, d.compat)` (5个兼容性字符串)
  - spec features → `__t('dev_feat_' + f, f)` (4个特性词)
  - spec note/spec text → `__t(d.spec.noteKey/SpecKey, ...)` 
- `renderProductCard()`:
  - `showDesc` → `__t('desc_'+p.id, showDesc)` (支持按 pid 翻译)
