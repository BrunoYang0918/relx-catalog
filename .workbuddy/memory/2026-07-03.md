## 改进 syncAll() 同步反馈体验

**问题**：同步数据到云端时，原有 toast 提示"同步完成：0/2"消失太快，用户体验差。

**修改内容**：
1. **CSS 新增**（index.html ~line 602）：添加 `.sync-progress`、`.sync-step`（`.done`/`.fail`/`.active`）、`.sync-final`、`.sync-dismiss` 样式
2. **renderSysAdmin() 修改**：在同步按钮下方添加 `<div id="syncProgress" class="sync-progress" style="display:none">` 作为持久进度区域
3. **syncAll() 重写**：
   - 使用 `results` 对象跟踪每个文件的同步状态
   - `renderAll()` 函数逐步渲染进度：已完成项（✅成功/❌失败）+ 未完成项（⏳等待）
   - 全部完成后显示汇总信息 + "确 定"按钮手动关闭
   - 不再依赖自动消失的 `showToast`

### 同步失败修复（commit `6c21c30` → `697fed1`）

### 多文件上传并发问题修复（commit `697fed1`）
- 问题：一次上传多个视频/图片时，每个文件产生一个 GitHub commit，导致 GitHub Pages 部署冲突，第一个部署常被取消
- 修复：媒体管理中的图片/视频上传改为 **单 commit 批量提交**（使用 GitHub Git Data API：创建 blob → 创建 tree → 创建 commit → 更新 ref）
- 效果：一次上传 N 个文件只产生 1 个 commit，只触发 1 次 GitHub Pages 部署
- 更新函数：`startVideoUpload()` 和 `startImageUpload()` 现在先读取所有文件，再统一调用 `uploadFilesBatchToGitHub()`
