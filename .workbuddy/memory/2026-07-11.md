## 🔴 重大架构发现（用户反馈"文件没有任何变化"根因）
- **运行时数据来自 JSON 配置文件，不是 index.html 硬编码！**
- 页面加载流程：硬编码 POD_SECTIONS(fallback) → localStorage缓存 → **`fetchRemoteJSON('pod-config.json')` 覆盖**(第2620行)
- `fetchRemoteJSON`按 hostname 分支：github.io→同域 / localhost/127.0.0.1/''→`./filename` 本地 / 其他→raw.githubusercontent.com
- **结论：只改 index.html 硬编码数据 → 运行时被旧 JSON 覆盖 → 改动不生效！**
- **修复规约：每次改产品数据后，必须同步重新生成 pod-config.json / disposable-config.json / device-config.json**
- 重新生成方法：Node 读 index.html，正则去掉`var X =`前缀 eval 三个对象字面量，JSON.stringify(obj,null,2) 写回对应 json
- 行号定位：POD_SECTIONS 1976-2045 / DISPOSABLE_SECTIONS 2060-2095 / DEVICES 2097-2104（会变，用 grep `^var XXX =` 和 `^};` 重新定位）
- `file:///` 打开会被浏览器 CORS 拦截 fetch → 用 `python -m http.server` 起本地服务通过 localhost 预览最可靠
- 本次已同步3个config并 bump DATA_VERSION v1.42，已本地commit未push
